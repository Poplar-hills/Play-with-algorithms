
寻找最短路径问题：
  - 应用：路径规划
    - 物理路径规划：运输网络
    - 虚拟路径规划：办公流程 —— 每个顶点代表一个任务，边代表该任务的资源花费，这样建模就可以用最短路径算法找出从起始任务开始，
      到达成最终目标之间所需完成的花费资源最少的中间任务。
  - 分类：
    - 无权图的最短路径：通过广度优先遍历（BFS）找到
    - 带权图的最短路径：通过松弛操作（Relaxation）找到
    - 图解：https://coding.imooc.com/lesson/71.html#mid=1494（4'25''）

在无权图中通过 BFS 寻找到的最短路径其实也是一种“最小”生成树，只不过：
  - 这里的“最小”指的是从起始顶点到图中所有其他顶点所经过的边的权值之和最小，而不是所有路径的权值之和最小。
  - 这种“最小”生成树也叫做“最短路径树”（Shortest Path Tree）。
  - 这种寻找最短路径树的问题叫做“单源最短路径问题”（Single Source Shortest Path Problem）。
